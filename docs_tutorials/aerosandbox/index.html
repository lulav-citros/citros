Aerosandbox

Overview​

The project is a simulation tool developed using ROS 2 (Robot Operating System) nodes and leverages the Aerosandbox Python library for aerodynamic calculations. Its primary objective is to simulate gliding flight scenarios for a Cessna 152 aircraft in the event of engine failure.

You can find more information about this useful aerodynamics library on Aerosandbox official website. All project installation, code overview and usage details are also available on the project's GitHub page.

Prerequisites​

  1. Please make sure you have all the necessary softwares to work with CITROS installed on your computer.
  2. Install Visual Studio code.
  3. We strongly recommend that you work with dockers. However, if you wish to work without dockers, please refer to the .devcontainer directory in project's repo, the dependencies you need are in the Dockerfile file.
  4. (Optional) Install Foxglove.

Installation​

  1. Clone the repository:

    git clone git@github.com:citros-garden/aerosandbox_cessna.git

  2. Open the repository in the VScode Dev Container.

  3. Build the project.

Workspace Overview​

This project contains only one launch file aerosandbox_cessna.launch.py. This file will be used for CITROS launch.

Scenario​

For this tutorial, let's check how far the Cessna can glide with engine failure depending on initial altitude. To find it out, we need to set up parameters and launch CITROS simulation.
The user provides flight parameters as input parameters to configure the simulation. These parameters are essential for defining the initial conditions of the simulated flight.
The simulation takes into account various flight dynamics and aerodynamic factors to model the gliding behavior of the Cessna 152. The maximum gliding distance depends on plane's aerodynamic parameters, initial altitude and initial velocity.
We will be using iPOPT - optimal under the hood problem solver (iPOPT is also installed as part of the AeroSandbox package).
The parameter setup is listed in .citros/parameter_setups/default_param_setup.json.
To find out how far the Cessna can glide with engine failure, we will examine the initial altitude, starting from 1000m and up to 10000m.

{
    "packages": {
        "aerosandbox_cessna": {
            "aerosandbox_cessna": {
                "ros__parameters": {
                    "h_0": {
                        "function": "my_func.py:func_with_context",
                        "args": [1000.0]
                    },
                    "v_0": 107.0,
                    "publish_freq": 10.0
                }
            }
        }
    }
}

The my_func.py file should contain:

def func_with_context(num, citros_context):
    return num + float(citros_context['run_id'])*1000

This function will set the h_0 parameter in range from 1000 to 1000+1000*n, where n = number of runs.

Learn more about parameter setup and defining custom functions in Directory parameter_setups and Adding Functions to Parameter Setup pages.

In addition to parameter setup, you can configure the simulation performance setup (timeout, CPU, GPU and Memory) as well. This parameters can be found in .citros/simulations/simulation_aerosandbox_cessna.json.
The default setup is 180 seconds timeout, 4 CPU, 4 GPU and 4096 MB of Memory.

Look in Directory simulations page for more information.

Running the Scenario Using CITROS​

Follow these steps to Run Simulation:

  1. Name your batch run simulation: aerosandbox_cessna
  2. Add a message to your batch run simulation: test run
  3. Run your simulation 10 times .

All the results will be saved under .citros/data/simulation_aerosandbox_cessna/[batch_name] folder.
in this case, batch_name is aerosandbox_cessna

To plot the results you can use Foxglove with the layout file existing in foxglove_layout folder.
