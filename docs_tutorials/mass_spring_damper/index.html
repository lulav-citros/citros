Mass Spring Damper

Overview​

This is a simple ROS demonstration of a mass-spring-damper system.

The example contains two ROS 2 packages: dynamics and controller.

System Dynamics​

The system's equations of motion:

mx¨=kf(t)−cx˙−kxm\ddot x = kf(t) -c\dot x -kxmx¨=kf(t)−cx˙−kx

and after laplace transformation (with zero I.C) we get a second order system:

XF=ωn2s2+2ωnζs+ωn2{X \over F} = {\omega_n^2 \over s^2 +2\omega_n\zeta s + \omega_n^2}FX​=s2+2ωn​ζs+ωn2​ωn2​​

where the natural frequency ωn=km\omega_n = \sqrt{k \over m}ωn​=mk​​

You can choose the system's parameters m, k and c and choose the initial condition x0, v0 and a0, all configured as ROS 2 parameters.

The Controller​

You can write your own controller to try stabilize the system for a given setpoint.

the default controller is a simple PID controller with the following form:

f(t)=kpe(t)+ki∫e(t)dt+kdddt(e(t))f(t) = {k_pe(t) + k_i\int{e(t)dt}} + k_d {d\over dt}(e(t))f(t)=kp​e(t)+ki​∫e(t)dt+kd​dtd​(e(t))

you can tune the controller gains, kpk_pkp​, kik_iki​, kdk_dkd​, configured as ROS 2 parameters.

Prerequisites​

  1. Please make sure you have all the necessary softwares to work with CITROS installed on your computer.
  2. Install Visual Studio code.
  3. We strongly recommend that you work with dockers. However, if you wish to work without dockers, please refer to the .devcontainer directory in project's repo, the dependencies you need are in the Dockerfile file.

Scenario​

Supposed we tune the PID gains of the controller for the nominal mass, and we reached a satisfying results.
Now we want to know how robust was the tuning for a normal distributed mass:

m=N(μ,σ)m = N(\mu, \sigma)m=N(μ,σ)

where:

μ=1.0,σ=0.3\mu = 1.0, \sigma = 0.3μ=1.0,σ=0.3

Learn more about parameter setup and defining custom functions in Directory parameter_setups.

The initial condition are:

r0=−1.0[m]v0=0.0[m/s]a0=0.0[m/s2]r_0 = -1.0[m]\\ v_0 = 0.0 [m/s]\\ a_0 = 0.0 [m/s^2]\\r0​=−1.0[m]v0​=0.0[m/s]a0​=0.0[m/s2]

With setpoint=0.0[m]setpoint = 0.0 [m]setpoint=0.0[m].

We will define the following requirements:

  • Maximum overshoot of 30%.
  • Settling time is 2.0 [sec].
  • Settling to 10% of the steady-state value.

Running the Scenario Using CITROS​

Follow these steps to Run Simulation:

  1. Name your batch run simulation: dynamics_controller
  2. Add a message to your batch run simulation: dynamics simulation
  3. Run your simulation 50 times .
  4. choose the simulation_dynamics_controller simulation

All the results will be saved under .citros/data/simulation_dynamics_controller/[simulation_name] folder.
in this case, simulation_name is dynamics_controller

Results​

  1. Create Database
  2. Upload data to the database
  3. Verify the data was loaded
  4. Execute the Notebook normal_distributed_mass.ipynb.
    You will find the notebook under citros_template/notebooks folder.

Now we can analyze the results:

We can see that 43 tests were passed the requirements, 6 failed and 1 was generated invalid mass (<0)(<0)(<0).
The maximum mass that still meeting the requirements is equal to 1.335[kg]1.335 [kg]1.335[kg]
